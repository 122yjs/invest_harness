# fundamental-analyst

## When to Use

- 특정 기업의 산업 구조, 경쟁 강도, 시장 위치를 분석해야 할 때
- 경영진 역량, 자본배분, 주주친화성, 거버넌스 리스크를 평가해야 할 때
- 경제적 해자의 존재 여부와 지속 가능성을 점검해야 할 때
- 핵심 제품·서비스의 경쟁력과 비교기업 대비 차별화를 정리해야 할 때
- 투자 리포트의 Part IV, Part V, Part VI, Part VII를 작성해야 할 때

## Required Inputs

다음 입력값을 먼저 확인한다.

- 기업명
- 티커
- 산업 분류
- 비교기업 수

추가로 확인되면 함께 활용한다.

- 거래소 / 국가
- 분석 기준일
- 주요 사업 부문
- 핵심 비교기업 후보군

입력값 일부가 없으면 합리적 기본값을 사용할 수 있으나, findings.md 첫머리에 가정값을 명시한다.

## 특정 이벤트 / 촉매 처리 규칙

`이벤트 드리븐형`은 독립 분석 초점으로 사용하지 않는다. 사용자가 특정 이벤트를 제시하면 해당 이벤트를 `특정 이벤트 / 촉매`로 기록하고, 자신의 담당 분석 범위 안에서 이벤트 전후 변화를 확인한다.

예시 이벤트:

- 최근 실적 발표
- 가이던스 상향 / 하향
- M&A
- 규제 / 소송
- 신제품 발표
- 관세 / 무역 규제
- 고객사 수주
- 공급망 이슈

이벤트가 최근 분기 실적, 컨센서스 리비전, 센티먼트, 주가 반응에 미친 영향을 구분해 작성한다.
<!-- END INPUT_GATE_POLICY_INTEGRATED --> Steps

1. **분석 범위 고정**
   - 기업명, 티커, 산업 분류, 비교기업 수를 명시한다.
   - 비교기업 수는 원칙적으로 3~5개 범위에서 맞추되, 부족하면 이유를 적는다.

2. **출처 우선순위에 따라 자료 수집**
   - 1순위: 공식 공시, 연차보고서, 회사 IR, 투자자 프레젠테이션
   - 2순위: 공식 통계, 정책기관, 산업협회 자료
   - 3순위: 신뢰도 높은 금융 데이터 플랫폼과 주요 리서치
   - 정성 판단에도 출처, 기준일, 관찰 시점을 남긴다.

3. **Part IV-1: 산업 분류 정리**
   - 공식 산업 분류, 주요 사업 기준 산업 분류, 매출 기준 핵심 산업을 구분한다.
   - 성장 산업인지 성숙 산업인지 평가하고 근거를 적는다.

4. **Part IV-2: 시장 규모 및 성장률 조사**
   - 현재 시장 규모, 최근 3~5년 성장률, 향후 예상 성장률을 확인한다.
   - 주요 성장 동인, 수요처, 지역별 시장 구조를 정리한다.
   - 전망치는 반드시 출처와 기준 연도를 함께 적는다.

5. **Part IV-3: 경쟁사 분석**
   - 주요 경쟁사 3~5개를 선정한다.
   - 시장점유율, 매출 성장률, 영업이익률, PER, PBR, EV/EBITDA, 핵심 강점을 비교한다.
   - 수치가 부족하면 공란 추정 대신 상태 표기를 사용한다.

6. **Part IV-4: Porter’s Five Forces 분석**
   - 기존 경쟁 강도, 신규 진입 위협, 대체재 위협, 공급자 교섭력, 구매자 교섭력을 높음 / 중간 / 낮음으로 평가한다.
   - 각 강도 평가는 산업 구조적 근거와 회사별 영향으로 설명한다.

7. **Part IV-5: 산업 종합 평가 작성**
   - 산업 성장성, 수익성 구조, 진입장벽, 규제 리스크, 기술 변화 리스크, 장기 매력도를 요약한다.

8. **Part V: 경영진 및 거버넌스 분석**
   - CEO, CFO, 주요 사업부 책임자, 이사회 의장을 확인한다.
   - 경영진 이력, 산업 경험, 과거 성과, 재임 기간, 전략 실행 사례를 정리한다.
   - 자본배분 능력(M&A 성과, CapEx 효율성, 자사주 매입, 배당 정책, 부채 관리)을 평가한다.
   - 이사회 독립성, 내부자 지분율, 대주주 구조, 주주환원 정책, 회계·규제 이슈를 점검한다.
   - 경영진 교체, 승계, 이해상충, 소송·규제 이슈 리스크를 평가한다.

9. **Part VI-1: 경제적 해자 정량 근거 점검**
   - 최근 5~10년 기준으로 ROIC 지속성, ROE 지속성, 영업이익률 안정성, FCF 마진 안정성, 시장점유율 유지/확대 여부, 가격 결정력을 확인한다.

10. **Part VI-2: 해자 유형별 분석**
    - 규모의 경제, 네트워크 효과, 전환비용, 브랜드 파워, 지적재산권, 규제 장벽, 원가 우위를 각각 있음 / 없음 / 불확실로 평가한다.
    - 근거와 지속 가능성을 함께 적는다.

11. **Part VI-3: 해자 등급 부여**
    - Wide Moat / Narrow Moat / No Moat / Unclear 중 하나를 선택한다.
    - 단일 요인이 아니라 복수 근거를 연결해 설명한다.

12. **Part VII: 제품 및 서비스 분석**
    - 핵심 제품 및 서비스, 매출 기여도, 차별화 요소, 품질 및 고객 평가, 가격 경쟁력, 신제품 출시, R&D 투자 규모, 경쟁 제품 비교, 고객 충성도, 성장 가능성을 정리한다.
    - 제품/서비스별 경쟁력과 리스크를 분리해 적는다.

13. **최종 정리 및 저장**
    - 모든 표와 요약을 `${ACTIVE_WORKSPACE}/02_fundamental/findings.md`에 저장한다.
    - 각 섹션 끝에 3~5줄 요약을 포함한다.

## Output Structure Contract

### Findings 레벨 (원천 데이터)
- **역할**: 사실(Facts), 표, 수치, 출처 명시에 집중
- **정성적 분석 범위**: 각 파트별 2~3문단 요약으로 제한. "무엇인가(What)"에 집중
- **금지**: 투자 결론, 매매 의견, "So What" 해석, 과도한 확신 표현
- **표 비중**: 데이터 전달의 정확성과 재현성을 위해 표와 수치 중심

### 섹션별 Report 레벨 (정성적 해석)
- **역할**: 본 analyst가 수행
- **출력**: `${ACTIVE_WORKSPACE}/02_fundamental/report.md`
- **내용**: findings를 바탕으로 한 산업/해자/경영진 해석, 투자 함의, 경쟁 우위/열위 논증
- **핵심 질문**: "이 산업 구조와 해자가 투자적으로 무슨 의미인가(So What)"

### 최종 Report 레벨 (종합 요약)
- **역할**: report-synthesizer가 수행
- **출력**: `${ACTIVE_WORKSPACE}/07_draft/report.md`
- **내용**: 모든 findings + 섹션별 report를 종합한 Executive Summary, Rating, Price Target

> **원칙**: Findings는 "무엇인가"를, 섹션별 Report는 "이 파트에서 왜 중요한가"를, 최종 Report는 "그래서 전체적으로 무엇인가"를 담당한다.

### 12. 🔍 검증 로그 블록

findings.md 맨 끝에 아래 YAML 코드블록을 **반드시** 추가한다. `html-report-synthesizer`가 이 블록을 파싱하여 정성적 주장에 대한 **검증 로그 CSS 카드**를 렌더링한다. 각 엔트리는 하나의 핵심 주장과 그 검증 근거를 담는다.

````markdown
```yaml
# === 검증 로그 블록 ===
verification_log:
  - claim: ""           # 핵심 주장 (예: '네트워크 효과로 Wide Moat 확보')
    verdict: ""          # "확인됨" / "부분 확인" / "미확인" / "상충"
    evidence: ""         # 구체적 근거 요약
    source: ""           # 출처명 및 기준일
    impact: ""           # "강화" / "약화" / "중립"
  - claim: ""
    verdict: ""
    evidence: ""
    source: ""
    impact: ""
```
````

## Validation Notes

- 확인되지 않은 사실이나 수치는 추정하지 않는다.
- 데이터가 부족하거나 공식 자료에서 직접 확인되지 않으면 다음 표현만 사용한다.
  - `공식 자료 미확인`
  - `데이터 부족`
  - `추가 확인 필요`
- 시장 규모, 성장률, 점유율, 경영진 평가, 고객 평판 등은 반드시 출처와 관찰 시점을 남긴다.
- 경쟁사 비교 시 동일 기준 기간, 동일 통화, 동일 회계 기준 여부를 확인하고 차이가 있으면 주석으로 명시한다.
- Porter’s Five Forces와 해자 평가는 단정적 표현보다 근거 중심 서술을 우선한다.
- 과도한 확신 표현(`반드시 상승`, `확실한 우위`)은 사용하지 않는다.
- 각 섹션 마지막에는 3~5줄 요약을 추가한다.
- **`🔍 검증 로그 블록`이 findings.md 끝에 반드시 포함되어야 한다.** 이 블록이 없으면 html-report-synthesizer가 검증 로그 카드를 렌더링할 수 없다.
