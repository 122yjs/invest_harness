# technical-analyst

## When to Use
- 투자 리포트의 Part IX. 기술적 분석을 작성할 때 사용한다.
- 최근 가격 흐름, 추세, 거래량, 기술지표를 통해 단기·중기 타이밍을 보조적으로 점검할 때 사용한다.
- 장기 투자 분석에서도 보조 신호로 지지선·저항선·추세 훼손 조건을 정리할 때 사용한다.
- 최종 산출물은 `${ACTIVE_WORKSPACE}/04_technical/findings.md`에 저장한다.

## Required Inputs
- 대상 기업명, 티커, 거래소
- 분석 기준일
- 가격 데이터 범위
  - 최근 1개월
  - 최근 3개월
  - 최근 6개월
  - 최근 1년
  - 최근 3년
- 가능하면 일봉, 필요 시 주봉 데이터
- 시가, 고가, 저가, 종가, 거래량
- 이동평균 계산값
  - 20일 이동평균
  - 60일 이동평균
  - 120일 이동평균
  - 200일 이동평균
- 기술지표 계산값
  - RSI 14
  - MACD 12, 26, 9
  - Bollinger Band 20, 2
  - ATR 14
  - 거래량 이동평균
  - OBV

## Workflow

<!-- BEGIN YFINANCE_MCP_TOOLS -->
### yfinance 활용 (전체 시장)

`yfinance_get_price_history`가 주력 도구로, OHLCV 데이터 + 기술적 차트(WebP)를 제공한다.

기술적 분석 순서:
1. `yfinance_get_price_history(symbol=..., period="1y", interval="1d")` → 일봉 데이터
2. `yfinance_get_price_history(symbol=..., period="1mo", interval="1h")` → 단기 시뇰
3. `yfinance_get_price_history(symbol=..., period="1y", interval="1d", chart_type="candle")` → 캔들 차트 이미지
4. `yfinance_get_price_history(symbol=..., period="1y", interval="1d", chart_type="vwap")` → VWAP 차트

획득한 OHLCV 데이터로 이동평균(20/60/120/200일), RSI(14), MACD(12,26,9) 등을 계산한다.

한국 기업은 korea-stock `get_stock_trade_info`로 KRX 공식 데이터를 1순위로 사용하고, yfinance는 차트 생성용으로 보조 사용한다.

### 2.2 MCP 도구 우선 사용 (한국 상장기업 한정)

korea-stock-mcp MCP 서버가 설치된 환경에서는 한국 상장기업 기술적 분석 시 아래 MCP 도구를 사용한다.

| 도구 | 데이터 | 활용 |
|---|---|---|
| `get_corp_code` | 종목코드 획득 | 6자리 코드로 KRX 도구에 전달 |
| `get_stock_trade_info` | 일별 종가, 등락률, 시고저, 거래량, 시총 | 추세, 지표 계산 |

사용 순서:
1. `get_corp_code`로 6자리 종목코드 획득
2. `get_stock_trade_info`로 기간별 주가 데이터 조회 (`basDd`: YYYYMMDD)
3. 복수 `basDd` 호출로 일별 데이터 누적 → 이동평균, RSI, MACD 계산

해외 주식의 경우 웹 검색으로 데이터를 수집한다.

## 특정 이벤트 / 촉매 처리 규칙

`이벤트 드리븐형`은 독립 분석 초점으로 사용하지 않는다. 사용자가 특정 이벤트를 제시하면 해당 이벤트를 `특정 이벤트 / 촉매`로 기록하고, 자신의 담당 분석 범위 안에서 이벤트 전후 변화를 확인한다.

예시 이벤트:

- 최근 실적 발표
- 가이던스 상향 / 하향
- M&A
- 규제 / 소송
- 신제품 발표
- 관세 / 무역 규제
- 고객사 수주
- 공급망 이슈

이벤트가 최근 분기 실적, 컨센서스 리비전, 센티먼트, 주가 반응에 미친 영향을 구분해 작성한다.
<!-- END INPUT_GATE_POLICY_INTEGRATED --> Steps
1. **분석 범위 확정**
   - 기준일, 사용 차트 주기(일봉/주봉), 데이터 기간, 가격 출처를 명시한다.
   - 장기 투자 맥락이면 기술적 분석은 보조 요소임을 분명히 적는다.

2. **가격 데이터 요약**
   - 최근 1개월, 3개월, 6개월, 1년, 3년 수익률 또는 가격 범위를 정리한다.
   - 신고가/신저가 여부, 변동성 확대 구간, 갭 발생 여부를 함께 본다.

3. **차트 및 추세 분석**
   - 주요 상승/하락 추세, 박스권 여부, 추세선 훼손 여부를 확인한다.
   - 눈에 띄는 지지선과 저항선을 가격대 기준으로 정리한다.
   - 단기와 중기 추세를 분리해서 쓴다.

4. **이동평균 분석**
   - 20/60/120/200일 이동평균 위치와 기울기를 확인한다.
   - 정배열/역배열, 골든크로스/데드크로스, 현재가와의 괴리를 정리한다.
   - 단순 교차보다 거래량과 추세 지속성까지 함께 해석한다.

5. **기술적 지표 해석**
   - RSI: 과매수/중립/과매도 구간과 다이버전스 여부를 본다.
   - MACD: 시그널선 교차, 0선 상하, 히스토그램 방향을 본다.
   - Bollinger Band: 상단/하단 접촉, 밴드 폭 확대·축소를 본다.
   - ATR: 절대 수준보다 최근 평균 대비 확대/축소를 해석한다.
   - 거래량: 추세 동행 여부, 돌파 신뢰도, 급증/급감 여부를 본다.
   - OBV: 가격과 함께 상승하는지, 분산 신호가 있는지 확인한다.

6. **매매 보조 시나리오 정리**
   - 주요 지지선, 주요 저항선, 손절 기준 후보, 돌파 확인 조건을 적는다.
   - 기술적 신호는 확률적 보조 도구이며 단독 투자 판단 근거가 아님을 명시한다.

7. **파일 저장**
   - 최종 결과를 `${ACTIVE_WORKSPACE}/04_technical/findings.md`에 저장한다.

## Output Structure Contract

### Findings 레벨 (원천 데이터)
- **역할**: 사실(Facts), 표, 수치, 출처 명시에 집중
- **정성적 분석 범위**: 각 파트별 2~3문단 요약으로 제한. "무엇인가(What)"에 집중
- **금지**: 투자 결론, 매매 의견, "So What" 해석, 과도한 확신 표현
- **표 비중**: 데이터 전달의 정확성과 재현성을 위해 표와 수치 중심

### 섹션별 Report 레벨 (정성적 해석)
- **역할**: 본 analyst가 수행
- **출력**: `${ACTIVE_WORKSPACE}/04_technical/report.md`
- **내용**: findings를 바탕으로 한 추세 해석, 지지/저항의 투자적 의미, 타이밍/리스크 신호
- **핵심 질문**: "이 기술적 패턴이 투자 타이밍으로 무슨 의미인가(So What)"

### 최종 Report 레벨 (종합 요약)
- **역할**: report-synthesizer가 수행
- **출력**: `${ACTIVE_WORKSPACE}/07_draft/report.md`
- **내용**: 모든 findings + 섹션별 report를 종합한 Executive Summary, Rating, Price Target

> **원칙**: Findings는 "무엇인가"를, 섹션별 Report는 "이 파트에서 왜 중요한가"를, 최종 Report는 "그래서 전체적으로 무엇인가"를 담당한다.

## Validation Notes
- 가격 데이터의 기준일, 수정주가 사용 여부, 일봉/주봉 여부를 명시한다.
- 이동평균과 지표 기간은 기본값(20/60/120/200일, RSI 14, MACD 12·26·9, Bollinger 20·2, ATR 14)을 유지하고 변경 시 이유를 적는다.
- 기술적 분석은 보조 신호다. 펀더멘털이나 밸류에이션을 대체하는 표현을 사용하지 않는다.
- 신호 해석은 단일 지표가 아니라 가격 추세, 거래량, 지지·저항과 함께 종합한다.
- `확실한 매수`, `반드시 반등` 같은 단정 표현을 사용하지 않는다.
- **`📊 대시보드 데이터 블록`이 findings.md 끝에 반드시 포함되어야 한다.** 이 블록이 없으면 html-report-synthesizer가 RSI 게이지, MACD 경고판, SMA 이격 상태, 지지/저항 밴드를 렌더링할 수 없다.
